# weekly-review

周复盘流程权威源。复盘完成后自动执行周归档，无需询问。

---

## 写入前置约束（强制 · 适用所有写入步骤）

凡步骤涉及写入流水/记录文件（MEMORY_LOG / ITERATION_LOG / 长期记忆.md / _本周.md / 候选池 / USER.md / persona_SOUL.md），**写入前必须先 Read 目标文件**，识别其 §条目模板 / §机制要点 / §条目格式 节，然后**严格按格式追加或修改**，不偏离不附加：

- **MEMORY_LOG §操作日志**：一行写完，不加括号附加说明；只记记忆代谢，架构变更不写此处
- **MEMORY_LOG §周复盘**：按文件 §条目模板 中的周复盘模板填充，字段不增不漏
- **ITERATION_LOG §变更记录**：按五字段（摘要/详情/影响/触发/关联）填充，摘要一句话，详情精简
- **候选池**（procedural/semantic_memory）：原地修改 ★ 数字；证据 / related / 日期 / 演化记录写入对应 `_archive/`；新条目按 `00.memory_agent.md §写入格式` 做主池 + archive + MEMORY_LOG 三处同步
- **USER.md / persona_SOUL.md**：按现有节结构追加，不破坏既有格式
- **长期记忆.md / _本周.md**：按既有节顺序约定追加（时间轴新→旧顶部插入；详细周录旧→新尾部追加；进展记录旧→新尾部追加）
- **写错位置即违规**

> 此约束目的：防止流水文件随时间膨胀变形。最容易膨胀的步骤（§5/§6/§7/§7.5）会在步骤内再点一次指针。

## 触发机制

| 方式 | 何时 | 谁触发 |
|------|------|--------|
| 自动触发 | 周日 daily-review 步骤6.5 | 告别时 daily-review 调用 |
| 启动提示 | 周日 week-sync 检测到未完成 | 会话开始时 week-sync 提示 |
| 手动触发 | 任意时刻 | 用户说触发词 |

> **每周日必须完成一次**。如果周日会话结束时还没有做，daily-review 步骤6.5 强制触发。
> 如果周日第一次开启新会话时还没做（_本周.md 无「本周产出」节），week-sync 主动提示。

## 复盘流程（严格顺序）

### 步骤 0 · 逻辑日期判定

读取当前物理时间。

- **物理 hour < 06:00** → 逻辑日期 = 物理日期 − 1，逻辑星期同步前移
- **物理 hour ≥ 06:00** → 逻辑日期 = 物理日期

后续所有写入字段统一用**逻辑日期**：
- 周次归属（W## 编号）按逻辑日期判定
- `_本周.md` 段落标题（凌晨工作写「周日延续 YYYY-MM-DD ~ MM-DD 凌晨」）
- `长期记忆.md §时间轴` 新行的日期范围
- `长期记忆.md §详细周录` W## 节标题
- `MEMORY_LOG.md §周复盘` 标题日期
- `_归档/2026-W##.md` 文件名按逻辑周次

**周一上午（hour ≥ 06:00）触发的特殊规则**：检查上一周复盘是否已完成（看 `MEMORY_LOG.md §周复盘` 是否有上周条目 + `_归档/` 是否有上周文件）。如果缺失，提醒用户补上周复盘——这种情况通常是周日彻夜跨过了 06:00 阈值。

### 步骤 1 · 全景摘要

根据 `_本周.md` 进展记录，生成本周完整全景报告：

**必须覆盖的维度**：
- **任务完成情况**：对照任务清单，哪些打勾完成，哪些未动，哪些新增
- **核心推进**：本周最重要的理论/研究/项目推进是什么（3-5 条，每条一句话）
- **关键产出**：本周新建或大幅更新的文件（不列清单，说有什么意义）
- **关键决策**：本周做出的框架性选择或方向判断
- **跨项目连接**：识别本周工作中跨项目的推理连续性（不要按项目切割独立汇报）
- **意外收获**：没有计划但实际完成或发现的重要东西
- **遗留未完成项**：本周没动或没完成的任务，是否需要延续到下周

向用户呈现这份全景后继续步骤2。

---

### 步骤 2 · 给用户空间

全景摘要报告完后，说：

「这是本周的全景。有没有什么特别重要的事情、转变、或者你自己的感受，你想先说说的？」

等待用户补充，将补充内容纳入后续步骤（不引导，不先问具体问题）。

---

### 步骤 3 · 主线对齐检查

对照 `长期记忆.md §当前处境`（主要矛盾 / 次要矛盾 / 大项目 / 转换预判），检查本周工作：

1. **对齐**：本周工作是否推进了当前主要矛盾？进了多少？
2. **偏移**：本周有没有大量时间花在主线以外的事（临时任务、旁路探索）？值不值？
3. **处境变化**：本周之后，长期记忆.md §当前处境 是否需要更新？
   - 有变化 → 覆盖更新（直接写入，S 级）
   - 无变化 → 跳过

---

### 步骤 3.5 · 情景模式提取（情景→schema 唯一系统通道）

**目的**：把本周情景流水中**跨日重复的内涵模式**显式提取为候选条目，进入候选池接受代谢。Execution Mode 的即时写入和 daily-review 的当日扫描是前两层兜底，weekly-review 是第三层。

**输入**：`_本周.md §进展记录` 全文（长期记忆.md §详细周录是本周摘要派生，不重复扫）+ 本周活跃项目 touched `_progress/` 段间转换标记（L2 `**开出**` / L2 `新问题` / L3 `> **问题承接**` / L3 `> **→ 开出的新问题**` / L3 `> **→ 本节收束**`）+ `MEMORY_LOG §周复盘暂存`（close-node 步骤 4.5 暂存的未达 schema 标准候选）+ 本周对话中 AI 脑内标记未入池的候选。

> 与 close-node 步骤 4.5 的关系：close-node 是节点级即时提取（达标即入池）；本步骤是周级跨节点去重 + 暂存复审 + 升星合并。两层互补，不重复——节点级已入池的条目本步骤只做升星 / 合并审查，不重新建。

**观察粒度判准（前置筛子）**：

| 类型 | 示例 | 处理 |
|---|---|---|
| **具体想法** | "用户认为 Loop Model v2.4 的 C1 应该是 X" | **不入候选池**，归项目主文档 / 情景层 |
| **内涵模式** | "用户在构建理论时先定义边界再收敛" | **可入候选池** P / S |

判断尺度：这条一年后还成立吗？依附具体内容的→不入；指向思路/方法论/行为模式的→入。

**扫描三问**（按层分类）：

| 层 | 扫描问 | 入池方向 |
|---|---|---|
| **P（程序层）** | AI 在本周被用户纠正过什么？是不是跨日跨场景的同类错误？ | `procedural_memory.md` |
| **S（语义层）** | 用户在思考/决策时，有没有显露出某种**思路/方法论/价值判断**的重复模式？ | `semantic_memory.md` |
| **E（情景层）** | 本周有没有阶段性结论或未决悬置需沉淀？ | 项目主文档 / 长期记忆.md §详细周录 · 不上升至候选池 |

**星级阶梯（同方向重复观察）**：

| 本周同方向观察次数 | 入池/升星 |
|---|---|
| 2 次 | ★（新建或现有条目保留 ★） |
| 3 次 | ★★（新建 ★★ 或现有 ★→★★） |
| 跨周再复现（已有条目本周再次命中） | 按 §M.2 升星规则 |

**去重**：每条候选在 `procedural_memory.md` / `semantic_memory.md` 现有条目中检索近似。找到 → archive 追加证据 + 主池升星；未找到 → 新建，按阶梯定星级，并三处同步。

**输出**：候选条目列表（N 级告知用户），在步骤 4 合并审查时一起处理。

---

### 步骤 4 · 候选池条目审查

对 `procedural_memory.md`（程序层）和 `semantic_memory.md`（语义层）全部条目逐一过审：

**激活检查**：
- 本周被引用、验证或强化的条目 → 重置衰减计时器（发现日期不改，但后续衰减从当前周复盘日起算）

**衰减检查**：高强度工作期距上次引用超过 2 周触发；非高强度期（度假 / 长低产期）先做工作强度评估，再按比例延后衰减周期。

| 当前星级 | 衰减处理 |
|---|---|
| **★★★★★** | 不在此流程（毕业触发，步骤 6 处理） |
| **★★★★** | 询问用户「这条记忆还成立吗？」（接近毕业，需谨慎，不强制降） |
| **★★★** | 询问用户「这条记忆还成立吗？」（不强制降） |
| **★★** | 静默降为 ★ |
| **★** | 询问「可以删除了吗，还是保留？」（删除/降无标记/保留） |

**新增候选合并**：
- 步骤 3.5 情景模式提取产出的新候选，在此并入正式流程
- daily-review 标记但尚未充分验证的 P/S 信号，在此决定是否升级强度

---

### 步骤 5 · 强度调整执行

根据步骤 3.5 + 步骤 4 的审查结果，统一执行强度调整：**定位条目原地修改 `**强度**：` / `**星级**：` 行的 ★ 数字；新观察（带日期）追加到对应 archive 条目；MEMORY_LOG 写一行代谢流水；不新建重复条目，不破坏其他字段**。逐条写入对应候选池文件与 archive（S 级，直接写入）。

> 写入前按本 skill §写入前置约束 先读候选池文件 §条目格式 节确认字段结构。

---

### 步骤 6 · 毕业检查

扫描两个候选池文件中所有已达 **★★★★★** 的条目。★★★★★ 为自动毕业触发强度。

**毕业路径**（按层分流）：

| 层 | 毕业目的地 | 说明 |
|---|---|---|
| **语义层**（偏好 / 认知框架 / 价值判断） | `<ASSISTANT_ROOT>/USER/USER.md` | 已是稳定用户身份事实，写入用户世界模型 |
| **程序层**（情境→行动模式） | `<ASSISTANT_ROOT>/SOUL/persona/persona_SOUL.md` | 已是稳定 AI 行为模式，写入人格身份层 |

**毕业是条目重写，不是搬运**。身份层只留净描述，log 元数据全部留在候选池 + 归档到 MEMORY_LOG §毕业归档。

对每条满足毕业条件的条目：

1. 判断毕业路径（语义层 / 程序层）
2. **判断写入节**：
   - 语义层 → `USER.md`：按条目内容性质分流（偏好/认知框架/方法论 → §认知模式；协作期待/沟通方式 → §协作注意事项；身份事实 → §核心身份）
   - 程序层 → `persona_SOUL.md §行为模式`（统一节，节内追加到末尾）
3. **C 级确认**：向用户报告毕业目的地（包括具体节）+ 剥壳后的净描述草案，逐条等待确认
4. 确认后**剥壳写入身份层**：
   - 程序层只写 **标题 + 情境 + 预期行动**（可加一句简述）
   - 语义层只写 **标题 + 情境 + 预期**（可加一句简述）
   - **剥掉**：星级 / 发现日期 / 来源 / 证据 / related / 备注等 log 字段
   - 按目标文件既有节结构追加，不破坏既有格式
5. 原候选池条目标 `graduated → [目的地] (YYYY-MM-DD)`，保留全部元数据不删，不参与衰减
6. **完整条目归档**：在 `MEMORY_LOG.md §毕业归档` 追加一条完整毕业条目（含原条目全部元数据 + 毕业日期 + 毕业去向 + 剥壳后的净描述副本）
7. 在 `MEMORY_LOG.md §操作日志` 追加一行指针：`YYYY-MM-DD | 毕业：[条目名] → [目的地]（详见 §毕业归档）`

> 写入前按本 skill §写入前置约束 先读 USER.md / persona_SOUL.md / MEMORY_LOG.md 确认目标节当前结构。

### 步骤 7 · 写入 MEMORY_LOG 周复盘条目

追加到 `MEMORY_LOG.md §周复盘` 节**尾部**（三级标题，从旧到新顺序，最新永远在文件最底）。本条目**只记记忆系统自身代谢**，工作进展不在此写（工作层写 长期记忆.md §详细周录）。

> 写入前按本 skill §写入前置约束 先读 MEMORY_LOG §条目模板 节中的周复盘模板，**字段不增不漏**，按以下模板逐字段填充：

```
### YYYY-WNN 周复盘（YYYY-MM-DD）

**本周记忆操作**：
- 语义层新增：[条目名称]（如无则「无」）
- 程序层新增：[条目名称]（如无则「无」）
- 情景→schema 提取：[本周从情景层提取升级为候选池条目的]（如无则「无」）
- 修正：[条目名称 原→新]（如无则「无」）
- 删除：[条目名称及原因]（如无则「无」）
- 毕业：[条目名称及目的地]（如无则「无」）

**强度变化**：
- 升级：[条目名称 ★→★★ 等]（如无则「无」）
- 降级：[条目名称 ★★→★ 等]（如无则「无」）

**衰减处理**：[处理了哪些触发衰减周期的条目（高强度期 2 周；非高强度期按工作强度延后），或「无」]

**复盘发现**：[本周记忆系统运行中发现的问题、模式、改进点，或「无」]

**下周迭代行动**：[记忆系统层面下周要做的调整，或「无」]

**参数调整**：[衰减周期/惊奇度阈值/星级阶梯等参数变动，或「无」]
```

### 步骤 7.5 · 写入 ITERATION_LOG（条件触发）

条件：本周发生以下任一架构级变更：
- 新增 skill 或大幅重写
- 工作流变更（步骤新增/删除/触发规则调整）
- 参数调整（衰减周期、写入规则等）
- 架构重构（文件结构/加载链变动）
- 协议迭代（写入/触发/确认规则改变）

满足 → 在 `ITERATION_LOG.md §变更记录` 节**尾部**追加版本化条目（从旧到新顺序，最新永远在文件最底），格式：`vX.Y.Z · 日期 · 类型`，包含摘要/详情/文件影响/触发/关联。版本号规则：主版本=破坏性变更，次版本=新功能/技能，补丁=修复/文档。

> 写入前按本 skill §写入前置约束 先读 ITERATION_LOG §条目格式 + §变更类型 节，按五字段模板严格填充，**摘要一句话，详情精简不水**。

不满足 → 跳过。

---

复盘流程全部完成后，立即执行归档操作：

**8a · 写入本周产出节**
在 `_本周.md §进展记录` 末尾（即所有日期节之后），追加（按 _本周.md 既有结构追加，不破坏 §任务清单 / §进展记录 顺序）：
```markdown
### 本周产出

[按项目/类别列出本周实际产出的文件，每条一行：`- 文件名.md — 说明`]
```

> 写入前按本 skill §写入前置约束 先读 _本周.md 当前结构。

**8b · 归档当周文件**
1. 将 `_本周.md` 重命名/移动为 `_归档/YYYY-Wnn.md`（Wnn 取 frontmatter 中的 `week` 字段值）
2. 在 `长期记忆.md §时间轴` **顶部插入**当周行（时间轴为 新→旧 索引表）：`| YYYY-Wnn | 日期范围 | 一句话概述 | [_归档/YYYY-Wnn.md] |`
3. 在 `长期记忆.md §详细周录` 节**尾部追加**当周详细周录节（详细周录为 旧→新 顺序，最新永远在文件底）：
   ```markdown
   ## Wnn · 详细周录（YYYY-MM-DD ~ YYYY-MM-DD）

   - **项目名1** — [本周核心推进，一段话] → `文件指针`
   - **项目名2** — [本周核心推进，一段话] → `文件指针`
   - **项目名3** — [本周核心推进，一段话] → `文件指针`
   ```
