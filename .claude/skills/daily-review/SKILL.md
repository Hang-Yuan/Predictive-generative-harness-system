# daily-review

对话结束汇总流程权威源。session_end.sh 检测到告别语后注入触发信号，AI 执行本流程。

---

## 输出规则

**所有输出以自然对话段落进行，不输出步骤编号、步骤标题、流程标签。** 步骤编号只是内部执行顺序参考，对话中不可见。

---

## 写入前置约束（强制 · 适用所有写入步骤）

凡步骤涉及写入流水/记录文件（MEMORY_LOG / ITERATION_LOG / 长期记忆 / _本周.md / 候选池 procedural_memory/semantic_memory），**写入前必须先 Read 目标文件**，识别其 §条目模板 / §机制要点 / §条目格式 节，然后**严格按格式追加**，不偏离不附加：

- **MEMORY_LOG §操作日志**：一行写完（`YYYY-MM-DD | [类型]：[简要描述]`），不加括号附加说明；只记记忆代谢，架构变更不写此处（应进 ITERATION_LOG）
- **ITERATION_LOG §变更记录**：按模板五字段（摘要/详情/影响/触发/关联）填充，摘要一句话，详情控制条数，不水
- **候选池**：按文件 §条目格式 字段填充（情境/预期/发现日期/证据/related，或情境/行动/预期后效/发现日期/证据/related），缺字段即违规
- **_本周.md / 长期记忆**：按目标节既有结构追加，不破坏顺序约定
- **写错位置即违规**（如架构变更进 MEMORY_LOG，记忆代谢进 ITERATION_LOG）

> 此约束目的：防止流水文件随时间膨胀变形。最容易膨胀的步骤（§6 / §6b）会在步骤内再点一次指针。

---

## 汇总流程（严格顺序，不可跳步）

### 步骤 0 · 逻辑日期判定

读取当前物理时间（timesense 注入或 Bash `date`）。

- **物理 hour < 06:00** → 逻辑日期 = 物理日期 − 1，逻辑星期同步前移
- **物理 hour ≥ 06:00** → 逻辑日期 = 物理日期

后续所有写入字段统一用**逻辑日期**：
- `_本周.md` 段落标题（如「周日延续 YYYY-MM-DD ~ MM-DD 凌晨」）
- `MEMORY_LOG.md §操作日志` 行首日期
- `ITERATION_LOG.md` entry 日期
- `_本周.md §本周任务` 完成日期标注

> 凌晨工作归本前一日的工作时段，不归物理新一天。timesense 注入的当前时间显示仍为物理时间，仅写入字段用逻辑日期。

---

### 步骤 1 · 回扫 P/S 信号 + 模式检测

#### 1a. P/S 信号写入

扫描完整对话记录，识别所有 P/S 类信号，然后写入：

- **P 类**（情境→行动模式）→ 写入 `MEMORY/procedural_memory.md`
- **S 类**（用户的认知框架/立场/工作方式/偏好/价值判断）→ 写入 `MEMORY/semantic_memory.md`

**写入前必须去重（每条信号都要检查）**：
1. 检查对应候选池文件是否已有内容相似的条目
2. **有相似条目** → 升级强度（★→★★→…→★★★★★），追加证据，不新建
3. **无相似条目** → 新建，初始强度 ★

**与 Execution Mode 的关系**：对话过程中 High 惊奇度的 P/S 条目已即时入池（★★★）。此步骤是**兜底扫描**——捡起对话中遗漏的 Medium 惊奇度信号，以及对已入池条目做去重/升星复核。

#### 1b. 模式检测（学习环）

扫描本次对话，识别重复操作模式（不限于纠正信号）：
- 某个查询/整理/格式化动作在本次对话中做了 2 次以上
- 或与现有程序层条目描述的情境高度匹配

匹配现有条目 → 对应条目升级一颗星。
新模式且不在候选池中 → 新建 ★ 条目。

#### 1c. 毕业候选标记

升级后检查：若任何条目达到 ★★★★★ → **不在此执行毕业**，仅在条目旁标注 `[毕业候选]`。

> 毕业执行权归 weekly-review 步骤 6（唯一入口）。daily-review 可提名、升星、标记候选，但不可写入 USER.md / persona_SOUL.md。

> 无信号则跳过。

---

### 步骤 2 · 回扫 E 类信号（C 级，汇总确认后写入）

扫描本次对话是否产出 E 类信号（情景层）。

**E 类信号判断标准**：结论确立 / 悬置点明确 / 阶段决策 / 框架更新

**如有 E 类信号**，生成确认表，等待用户确认后执行写入：

| 信号 | 内容摘要 | 建议写入位置 | 确认？ |
|------|---------|-------------|--------|
| [结论/悬置/决策/框架] | [一句话描述] | [项目主文档 §节名 / 研究记录 / 文献记录] | ✓ / 跳过 |

等待用户确认。用户说「跳过」或不回应则整体跳过。

**E 类写入位置**（由当前工作上下文决定）：
- 当前在 `01 项目区/[X]` → 该项目主文档对应节
- 当前在 `02 阅读区/[Y]` → 该笔记 / 读书记录文件
- 当前在 `03 写作区/[Z]` → 该作品对应节
- 跨上下文通用结论 → 长期记忆.md §当前处境 或 `_本周.md`

**跨项目指针（C 级确认）**：若某条 E 类结论同时影响多个项目，写入权威落点后，在确认表中额外列出受影响的其他项目，等用户确认后在这些项目文档中添加指针（`→ 权威源：[项目X/主文档 §节名]`）。不复制结论内容，只写指针。

**禁止未经确认直接修改项目文件。**

无 E 类信号则跳过。

---

### 步骤 3 · 更新 长期记忆.md 当前处境

条件：本次对话中主要矛盾或项目状态有实质变化。

有变化 → 覆盖更新 `长期记忆.md §当前处境`（S 级，直接写入）。
无变化 → 跳过。

---

### 步骤 3.5 · 节点闭合兜底扫描

扫描本次对话，识别"已事实上闭合但未经 close-node 流程处理"的节点——即某段工作/讨论已收束（结论落定 / 决策拍板 / 任务完成）但主文档 / `_overview.md` / `_progress/` / 指针链未同步。

- 有 → 按 `close-node` skill 流程逐节点补跑（主文档 / overview / progress / 指针 / 任务勾选 / 悬置）
- 无 → 跳过

> 权威源：`~/.claude/skills/close-node/SKILL.md`。本步骤是长工作日兜底——close-node 正常走节点级即时触发，此处只捡漏。

---

### 步骤 4 · 写入 _本周.md 今日进展

在 `_本周.md §进展记录` 追加今日工作段（S 级，直接写入）。

**顺序约束**：日期节按时序从上到下追加，最新日期永远在 §进展记录 末尾；同日内多个工作段按发生时间从上到下追加。

**日期节**：找到或新建今日日期节 `### 周X YYYY-MM-DD`。

**每个工作段格式**（有几个项目写几段）：

```markdown
**项目名称**（`项目根目录路径`）
关联：`具体文件1.md` · `具体文件2.md`

- 今天做了什么（流水账，不记理论推导细节）
- 确立了什么决策
- 完成了什么，推进到哪里
```

> `关联：` 字段是接续检查的锚点，必须准确列出本次实际修改或深度阅读的文件名。

**归档日额外检查**：若当日为归档日（周日 / 明确触发归档），确认 `### 本周产出` 节：
- 缺失或位置不在进展记录末尾（所有日期节之后） → 立即生成并追加

### 步骤 6 · 写入 MEMORY_LOG.md（条件触发）

条件：本次会话发生**记忆系统代谢**：候选池条目变更（新增 / 修正 / 删除 / 升星 / 降星）

满足 → 在 `MEMORY_LOG.md §操作日志` 节**尾部**追加一行（从旧到新顺序，最新永远在文件最底），格式：`YYYY-MM-DD | [类型]：[简要描述]`，**一行写完，不加括号附加说明**。
不满足 → 跳过。

> 写入前必须按本 skill §写入前置约束 先读 MEMORY_LOG §条目模板 + §机制要点 节。
> 架构/技能/协议/人格变更**不写这里**，见步骤 6b。误写即违规。

---

### 步骤 6b · 写入 ITERATION_LOG.md（条件触发）

条件：本次会话发生以下任一**架构级变更**：
- 系统架构变更（文件结构 / 协议规则 / 继承链 / 加载链）
- 技能创建 / 重写 / 废弃
- 人格文件变动
- 记忆系统协议迭代
- Hook 脚本重写

满足 → 在 `ITERATION_LOG.md §变更记录` 节**尾部**追加版本化条目（`vX.Y.Z · 日期 · 类型`，含摘要/详情/影响/触发/关联；从旧到新顺序，最新永远在文件最底）。版本号：主=破坏性 / 次=新功能 / 补丁=修复。
不满足 → 跳过。

> 写入前必须按本 skill §写入前置约束 先读 ITERATION_LOG §条目格式 + §变更类型 节，按五字段模板严格填充，摘要一句话，详情精简。

---

满足 → 执行 `weekly-review` skill 完整流程。

> 权威源：`~/.claude/skills/weekly-review/SKILL.md`

不满足 → 跳过。
