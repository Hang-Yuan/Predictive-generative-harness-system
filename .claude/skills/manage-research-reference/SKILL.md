# manage-research-reference

科研类项目的文献管理 skill。一条铁律：**研究记录 / 项目主文档行内只写简标 `(Author, Year)`；完整文献信息一律集中到 `_reference/文献记录.md`，按主题分节**。

## 加载链

**上游**：
- `create-project` skill — 项目类型为理论研究 / 系统工程，且用户首次引用文献时调用
- `write-progress` skill — `_progress/` 子文件追加新文献引用时调用
- AI 自驱 / 用户直接调用 — 任何"补一条文献到记录"场景

**下游**：
- `01 项目区/[项目]/_reference/文献记录.md` — 文献集中索引（按主题分节）

**同级联动**：
- `new-file/SKILL.md` — 首次建 `文献记录.md` 时强制补加载链

---

## 核心约定

1. **行内引用极简**：研究记录 / 主文档里只写 `(Zacks et al., 2002)` / `(PMC2948259)` 等简标；不写期刊 / DOI / 摘要
2. **集中索引**：完整文献信息（作者 + 年份 + 标题 + 来源 URL + 支撑位置 + 核心内容）一律落 `_reference/文献记录.md`
3. **按主题分组**：`## 主题` / `### 子主题`；不按时间排，不按字母排——按论证用途分组
4. **支撑位置必填**：每条文献必须标注它在哪个主文档 §节被引用（`项目主文档.md §节名` 或 `_progress/[文件].md §节`）；多处引用列多行
5. **来源可回溯**：URL 必填（PubMed / PMC / arXiv / 期刊页 / 出版社页 / Google Scholar 任一可访问入口）

## 操作流程

### 场景 A · 首次建立 `_reference/文献记录.md`

触发：用户在研究记录或主文档首次引用一篇文献，且项目下还没有 `_reference/` 容器。

1. 建项目下 `_reference/` 目录（mkdir）
2. 调用 `new-file` skill 检查加载链字段
3. 按上方"文件结构"骨架写入 `文献记录.md`，含 frontmatter + 加载链 + 引言指针
4. 选定主题分节并写入第一条文献条目
5. 同步更新项目 `_overview.md`：
   - §管辖文件（下游） 追加 `_reference/文献记录.md`
   - §项目文件 / §子目录 标注 `_reference/` 已建
6. 提示用户行内引用按简标格式（`(Author, Year)` 或 `(PMID/PMC ID)`）

### 场景 B · 追加新文献到现有记录

触发：研究推进中引用了新文献，且项目已有 `_reference/文献记录.md`。

1. 读 `文献记录.md` 全文，识别现有主题分节
2. 判断新文献归属：
   - 命中现有主题 → 追加到该 §节末尾
   - 不命中任何主题 → 评估是否新建主题节（需要 ≥2 条相关文献才值得新建一节，否则先归"杂项"或最近主题）
3. 写入条目：作者(年份) + 标题 + 来源 URL + 支撑位置 + 核心内容
4. 更新 frontmatter `updated` 字段
5. 提示用户行内引用用简标

### 场景 C · 与检索 agent（如 general-search-agent）整合

触发：通过检索 agent 检索文献后，需要将结果纳入 `文献记录.md`。

1. 检索结果中只筛保留**支撑当前论证或开出新问题的**条目，不全量入档
2. 按场景 B 写入流程追加
3. 在 `_progress/[相关文件].md` 对应节使用简标 `(Author, Year)` 行内引用
4. 检索的废弃 / 不相关条目不入档（避免文献噪音）

## 与相邻 skill 边界

| skill | 职能 |
|---|---|
| **manage-research-reference**（本） | 文献管理 / `_reference/文献记录.md` 写入 / 行内引用约定 |
| `create-project` | 建项目入口；本 skill 由它在用户首次需要文献时调用 |
| `write-progress` | 推进过程写入；行内引用简标由本 skill 约定，但在 `_progress/` 文件里使用由 write-progress 落地 |
| `new-file` | 首次建 `文献记录.md` 时强制补加载链 |

---

## 不做

- 不替代行内引用——简标写在 `_progress/` / 主文档里仍归 write-progress / 用户自己写
- 不做正式学术格式化——本 skill 是工作记录，不是论文 References；正式文献格式化在投稿前另行处理
- 不批量重写既有 `文献记录.md`——只对新条目 / 新建场景作用
- 不去重检索结果——筛选交给检索 agent 调用方判断

---

## 注意

- **主题分组按论证用途，不按文献本身属性**：同一篇 Zacks 2002 在 [项目 1 占位] 里可能进 "C1 — 全局工作空间"，在另一个项目里可能进别的主题——按**它在本项目里支撑什么**分组
- **支撑位置失效要修正**：主文档章节改名 / 节合并 / 节删除时，支撑位置字段需要同步改，否则索引断链
- **不预建空主题**：场景 A 首次建立时只写第一条文献和它所在主题；后续主题随文献追加自然展开
