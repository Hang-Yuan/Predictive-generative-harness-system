## Overview

**Your Projects** stores all active or completed project materials. Each project gets its own folder, self-contained.

Core logic: **project isolation**—different projects don't share information by default unless you explicitly request cross-referencing.

## Rules

### Project Isolation

| Rule | Description |
|------|------------|
| Default open | Projects under same user default to allowing cross-reference |
| Explicit isolation | Confidential projects: add `isolation: true` to `_概览.md` YAML frontmatter |
| Deliverable protection | Deliverables involving clients/partners: confirm before modifying |

### Project Lifecycle

```
Create → Active → Wrap-up → Archive
```

### Operations

| Operation | Permission |
|-----------|-----------|
| Read project files | Free |
| Create new files in project | Free |
| Modify project docs | State what changes |
| Delete files | Requires user confirmation |
| Cross-project reference | Requires user authorization |
| Archive entire project | Requires user confirmation |

### AI Behavior

- **Before entering any project, read its `_概览.md` first** (if exists)
- Within current project, proactively search related files
- Don't carry Project A details into Project B conversations
