## 区域定位

**Projects 区域** 存放所有进行中或已完成的项目材料。一项目一文件夹，自成体系。

核心逻辑：**项目隔离** —— 不同项目默认不互通，除非你明确要求交叉引用。

---

## 目录结构

```
01 Projects/
├── 00.projects_agent.md      ← 本文件
├── [项目A]/
│   ├── _概览.md               ← 项目总览（标准入口，必备）
│   ├── [设计文档]
│   └── [交付物]
├── [项目B]/
└── _归档/
```

---

## 规则

### 项目隔离

| 规则 | 说明 |
|------|------|
| 默认开放 | 同一用户名下的项目默认允许交叉引用 |
| 明确隔离 | 涉密项目：在 `_概览.md` YAML frontmatter 中加 `isolation: true` |
| 交付物保护 | 涉及客户/合作方的交付物：修改前必须确认 |

### 项目生命周期

```
创建 → 活跃 → 收尾 → 归档
```

### 操作权限

| 操作 | 权限 |
|------|------|
| 读取项目文件 | 自由 |
| 在项目内新建文件 | 自由 |
| 修改项目文档 | 说明改动内容 |
| 删除文件 | 需用户确认 |
| 跨项目引用 | 需用户授权 |
| 归档整个项目 | 需用户确认 |

### AI 行为约束

- **进入任何项目前，先读它的 `_概览.md`**（如存在）
- 在当前项目内，主动搜索相关文件
- 不要把项目 A 的细节带到项目 B 的对话里
