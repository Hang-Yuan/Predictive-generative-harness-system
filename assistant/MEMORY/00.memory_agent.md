# 记忆区 · 区域 agent

候选池的操作细则。宏观规则（记忆层架构 / 惊奇度 / 路由 / 毕业路径）在 `CLAUDE.md §M`，本文件是**执行级增量**：条目格式模板、代谢操作步骤、判断锚点、版本记录规则。

## 加载链（上下游）

- **管辖文件（下游）**：
  - `procedural_memory.md` — 程序层候选池（条目）

## 写入格式（v3.2 · 主池 + archive）

所有新条目统一用 schema 格式。主池只保留运行时需要常驻注入的轻字段；证据 / related / 日期 / 演化记录外迁到 `_archive/`。

### 程序层（procedural_memory.md）

```
### [模式名称]
- **强度**：★ / ★★★ / ★★★★★
- **触发情境**：[触发条件——什么情况下该模式被激活]
- **行动模式**：[AI 应该做什么]
- **预期后效**：[执行后预期用户/世界如何反应]
- **来源指针**：`_archive/procedural_archive.md §[条目ID]`
```

### 语义层（semantic_memory.md）

```
### [标题]
- **强度**：★ / ★★★ / ★★★★★
- **情境**：[何时此 schema 被激活]
- **预期**：[在此情境下用户/世界应该如何]
- **来源指针**：`_archive/semantic_archive.md §[条目ID]`
```

### archive 详记（_archive/*.md）

```
### [条目ID] · [模式名称]
- **主池条目**：`procedural_memory.md §...` / `semantic_memory.md §...`
- **发现日期**：YYYY-MM-DD
- **证据**：[原始观察依据 / 对话锚点 / progress 段间转换]
- **related**：[关联条目，无则 —]
- **演化记录**：[new / up / mod-情境 / mod-预期 / 合并 / 毕业 / 弃用]
```

### 兼容说明

本版开始采用字段外迁格式；既有条目迁移时，主池保留运行字段，证据只追加到对应 archive。

## 判断锚点（程序 vs 语义）

- 落点是「AI 遇 X 做 Y」 → **程序层**
- 落点是「用户是这种人 / 用户持这种立场 / 用户这样工作和思考」 → **语义层**
- 身份层冲突（直接抵触 USER / persona_SOUL）→ 不动身份层，作为 High 惊奇度入对应候选池

## 代谢操作

完整规则见 `CLAUDE.md §M.2 §M.3`。本节是执行步骤：

### Schema 比对动作（Execution Mode 即时）

| 结果 | 信号 | 动作 |
|---|---|---|
| 命中身份层 + 符合 | Low | 不动 |
| 命中身份层 + 相反 | High | 不动身份层，入候选池走修正 / 升星路径 |
| 命中候选池 + 符合预期 | Low | 直接升星（稳定性证据，不新建） |
| 命中候选池 + 偏离预期 | High | 修正情境边界 / 新增例外 / 拆条，修正后升星 |
| 未命中 | Medium | 新建 ★ schema 条目 |

所有写入动作必须三处同步：主池写 schema 本体，`_archive/` 写证据详记，`MEMORY_LOG.md` 写一行代谢流水。

### 升星（同方向再次观察）

在原条目上把星级 `★` → `★★`（依此类推）。证据追加到对应 archive 条目，不写回主池。不重复建条目。

### 降星 / dormant（Review Mode 批量）

- schema 连续 N 周无命中 → 降一星
- 降至 0 星 → 标 `[dormant]`
- dormant 后再无命中 → 删除

### 毕业（★★★★★）

**毕业是条目重写，不是搬运。** 身份层（USER / persona_SOUL）是稳态规则，不存日志元数据；MEMORY_LOG 才是审计流水落点。

**四步走**：

1. **剥壳写入身份层**：只搬净描述字段——
   - 程序层：**标题 + 情境 + 预期行动**（可加一句简述"为什么这样做"）
   - 语义层：**标题 + 情境 + 预期**（可加一句简述）
   - **剥掉**：星级 / 发现日期 / 来源 / 证据 / related / 备注等 log 字段
   - 程序层写入 `SOUL/persona/persona_SOUL.md §行为模式`，语义层写入 `USER/USER.md`（节分流见 weekly-review §步骤 6）
2. **原条目标毕业**：候选池原条目标 `graduated → [目的地] (YYYY-MM-DD)`，保留全部元数据不删，不参与衰减
3. **完整条目归档到 MEMORY_LOG**：在 `MEMORY_LOG.md §毕业归档` 追加一条**完整毕业条目**（含原条目全部 log 元数据 + 毕业日期 + 毕业去向 + 剥壳后的净描述副本），这是毕业事件的唯一完整审计记录
4. **§操作日志加一行指针**：`YYYY-MM-DD | 毕业：[条目名] → [目的地]（详见 §毕业归档）`

**触发条件**：weekly-review 步骤 6 扫描到 ★★★★★ + 跨周稳定 + C 级确认，不自动触发。

### 冲突

新条目与已有条目矛盾 → **不静默覆盖**，N 级告知用户，等裁决。

## Bloat 阈值

`procedural_memory.md` 和 `semantic_memory.md` 各 ≤30 条（合计 ≤60）。详见 `CLAUDE.md §T`。

- 单文件超 30 条 → 优先触发 weekly-review 深度代谢（升星毕业 / 降星衰减 / 合并），不扩容
- 合计超 60 条 → 强制代谢，必要时手动拆分（如 `semantic_偏好.md` / `semantic_框架.md`）
