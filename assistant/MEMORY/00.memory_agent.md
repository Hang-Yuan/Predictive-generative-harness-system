- Falls under "AI encounters X does Y" → **Procedural layer**
- Falls under "User is this kind of person / holds this position / thinks and works this way" → **Semantic layer**
- Identity layer conflict (directly contradicts USER / persona_SOUL) → don't modify identity layer, enter candidate pool as High surprise

## Metabolism Operations

Full rules in `CLAUDE.md §M.2 §M.9`. This section is execution steps:

### Entry (Execution Mode real-time)

| Surprise | Stars | Action |
|---|---|---|
| High (match + deviant) | ★★★ | Upgrade existing schema or add new exception entry |
| Medium (no match) | ★ | Create new schema entry |
| Low (match + expected) | — | Don't write |

### Star Upgrade (same-direction re-observation)

On original entry, upgrade `★` → `★★` (and so on), append "re-observed · date" to evidence section. Don't create duplicate entries.

### Star Downgrade / Dormant (Review Mode batch)

- Schema no match for N consecutive weeks → downgrade one star
- Down to 0 stars → mark `[dormant]`
- Dormant with continued no match → delete

### Graduation (★★★★★)

- Procedural ★★★★★ → migrate to `SOUL/persona/persona_SOUL.md §Behavior Patterns`, mark original `graduated → SOUL §Behavior Patterns`
- Semantic ★★★★★ → migrate to `USER/USER.md`, mark original `graduated → USER`
- Graduation requires weekly-review confirmation + cross-week stability, not auto-triggered
- After graduation, update MEMORY_LOG.md + log one line

### Conflict

New entry contradicts existing entry → **don't silently overwrite**, N-level inform user, wait for ruling.
