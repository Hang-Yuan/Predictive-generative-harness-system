## 加载链（上下游）

**上游**：`CLAUDE.md §M · 记忆系统` — 记忆系统宏观规则入口。

**管辖文件（下游）：**

## 核心边界

记忆系统只记录会改变未来预测或行动的信号。事实本身不构成写入理由。

三类运行文件：

| 文件 | 启动注入 | 职责 |
|---|---|---|
| `episodic_inbox.md` | 否 | 实时追加校准信号。 |
| `episodic_memory.md` | 否 | 保存 1-3 星的情景级候选 schema。 |
| `semantic_memory.md` | 是，最多 8 条 | 保存 4-6 星的语义级 schema，参与运行时预测。 |

身份层：

| 位置 | 职责 |
|---|---|
| `USER/USER.md` + 子文件 | 用户长期稳定特质。 |
| `SOUL/persona/persona_SOUL.md` | AI 长期稳定行为模式。 |
| `~/.claude/skills/` | 流程化、可执行的稳定行动规则。 |

默认动作是遗忘。只有被预测关系和用户校准信号反复支持的模式，才允许上行。

## 两轴判定

每次观察先判断两个问题：

1. **P轴：预测关系**。这次观察与当前 schema 预测是什么关系？
2. **C轴：校准信号**。用户是否给出了明确校准？

### P轴

| 值 | 定义 |
|---|---|
| 命中 | 观察符合当前 schema 的预测。 |
| 击穿 | 观察反向校准或否定当前 schema 的预测。 |
| 未覆盖 | 当前 schema 没有覆盖这类观察。 |

### C轴

| 值 | 定义 |
|---|---|
| 正向确认 | 用户明确赞同、确认、满意或强化。 |
| 反对 | 用户明确反对、不满或拒绝。 |
| 纠正 | 用户修正某点，但非全盘否定。 |
| 中性 | 用户没有给出明确校准。 |

### 写入判定

```text
P轴=命中 且 C轴=中性 -> 不写
其他组合 -> 写 episodic_inbox
```

`命中 + 正向确认` 要写；它是稳定 schema 的强化证据。

## 写入授权

| 目标 | 写入者 | 边界 |
|---|---|---|
| `episodic_inbox.md` | 主模型按 hook 提示自判后追加 | 只写一行校准信号，不写长期结论。 |
| `episodic_memory.md` | close-node / daily-review / weekly-review | 写情景级候选，不启动注入。 |
| `semantic_memory.md` | weekly-review 为主 | 写语义级 schema，控制启动注入上限。 |
| `USER.md` / `SOUL.md` / skill | [用户称呼] C verdict 后写 | 只写剥壳后的稳定描述或流程。 |

物理删除文件或历史证据属于 C 级。

## 证据充分度

提取深度由证据充分度决定，不由工具触发时点决定。

| 场景 | 可做动作 |
|---|---|
| 实时对话 | 写 `episodic_inbox.md`。只有显式加载了足够历史材料时，才可提出更深提取。 |
| close-node | 聚合当前节点内多次校准，生成或更新 `episodic_memory.md` 候选。 |
| daily-review | 聚合当日多节点校准，执行日级衰减。 |
| weekly-review | 处理跨日 / 跨周稳定性，执行 semantic 升降、合并、毕业候选。 |

项目 In_Progress 是证据源，不是记忆池。项目结论留在项目主文档；从项目推进中提取的只能是不依赖该项目的可迁移模式。

## episodic_inbox 规则

写入位置：文件末尾。

写入方式：使用 Bash 追加，例如：

```bash
cat >> "<ASSISTANT_ROOT>/MEMORY/episodic_inbox.md" <<'EOF'
| JYYYYMMDD-001 | YYYY-MM-DD HH:mm | 对话 | 击穿 | 纠正 | 混合 | schema 指针 | 一句话观察 | 活动 |
EOF
```

禁止使用 Edit 改写整份文件；禁止插入文件中段。

保留期：未被升格的条目保留 7 天；到期由 daily-review 或 weekly-review 标记为已衰减。

## episodic_memory 规则

`episodic_memory.md` 保存 1-3 星候选。条目是当前程序型 / 语义型候选的统一形态，用 `类型` 字段区分。

### 生成

触发：close-node 或 daily-review。

动作：

1. 读取相关 `episodic_inbox` 条目。
2. 按预测对象、校准信号、情境域聚合。
3. 判断是否可写成可复现的“情境 -> 行动 / 预期”。
4. 可复现则生成或更新 `episodic_memory` 条目。
5. 原 `episodic_inbox` 条目标记为已消耗。
6. 未升格条目按保留期衰减。

### 字段

| 字段 | 说明 |
|---|---|
| 强度 | ★ / ★★ / ★★★ |
| 状态 | 活动 / 复审 / 已归档 |
| 类型 | 程序 / 语义 / 混合 |
| 创建日期 | 条目首次生成日期。 |
| 最后激活 | 最近一次被用户校准、命中、提取或复用的日期。 |
| 提取次数 | 被 weekly-review 提取为 semantic 或用于 semantic 复审的次数。 |
| 触发情境 | 该候选在哪类情境下被激活。 |
| 行动模式 | 程序型或混合型条目需要写；模型应如何行动。 |
| 预期 | 语义型或混合型条目需要写；未来应如何被预测验证。 |
| 来源指针 | 指向 inbox、bus、progress 或 archive。 |

### 升降

| 条件 | 动作 |
|---|---|
| 7 天内再次被用户校准 | 强度 +1；更新最后激活 |
| 被明确击穿 | 重写情境 / 行动 / 预期，或新建竞争候选 |
| 连续 7 天无命中 | 强度 -1；状态可转复审 |
| 降到 0 星 | 状态改已归档，不物理删除 |
| 强度达到 ★★★ 且通过等价 schema 检查 | weekly-review 执行 semantic 升格；提取次数 +1 |

## semantic_memory 规则

`semantic_memory.md` 只保存启动需要注入的语义级 schema。主文件保持极简；证据全部写入 `_archive/semantic_archive.md`。

### 升格

触发：`episodic_memory` 候选达到 ★★★ 且通过等价 schema 检查。

动作：

1. 检查 `semantic_memory.md` 是否已有等价 schema。
2. 若已有等价 schema：合并到既有条目，证据写入 archive。
3. 若无等价 schema：新增语义记忆条目，证据写入 archive。
4. 若启动注入组超过 8 条，按排序规则将最低优先级条目降回 `episodic_memory.md`。

等价判准：未来遇到同类触发时，会给出同一预测、同一纠偏方向、同一行动约束。

### 主文件字段

| 字段 | 说明 |
|---|---|
| 强度 | ★★★★ / ★★★★★ / ★★★★★★候选 |
| 状态 | 启动注入 / 备用 / 需要修订 / 毕业候选 |
| 类型 | 程序 / 语义 / 混合 |
| 预测情境 | 何时调用这条 schema。 |
| 行动预期 | 模型应如何预测或行动。 |
| 证据指针 | 指向 `_archive/semantic_archive.md` 对应条目。 |

### 排序

启动注入优先级：

1. 高风险行为约束。
2. 强度更高。
3. 最近 14 天命中。
4. 与当前处境或本周任务相关。
5. 未被身份层稳定覆盖。

### 衰减

| 条件 | 动作 |
|---|---|
| 整周无命中 | 强度 -1 |
| 被击穿并纠正 | 标 `需要修订`；weekly-review 重写或降级 |
| 降到 3 星 | 移出启动注入，写回 `episodic_memory.md` |
| 达到 6 星且跨周稳定 | 生成毕业候选，等待 C verdict |

## 毕业

毕业条件：

```text
6星 + 跨周稳定 >= 3 周 + [用户称呼] C verdict
```

高风险纪律可在 6 星 + 2 周稳定时提出候选，但仍需 C verdict。

毕业流程：

1. weekly-review 只生成候选。
2. 候选写明类型：程序 / 语义 / 混合。
3. 候选写明推荐去向：SOUL / USER / skill。
4. [用户称呼] C verdict 后才写正式目标。
5. 写入前剥壳：去星级、去临时证据、去现场情绪、去重复来源，只保留稳定描述和必要边界。
6. 写入后，`semantic_memory.md` 条目标记已毕业并移出启动注入。

混合类型优先拆分；不能拆时只保留一个权威源，其他位置写指针。
