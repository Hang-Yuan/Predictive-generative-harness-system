# 记忆系统日志

## 加载链（上下游）

**上游（由谁加载本文件）：**
不在启动序列必须层。按需加载（审查记忆操作历史时）；daily-review 步骤 6 / weekly-review 步骤 7 时写入。启动序列亦可读尾部 N 行了解近期代谢节奏。

**同级联动：**
- `00.memory_agent.md` — 写入格式 / 代谢规则 / 判断锚点（规则权威）
- `procedural_memory.md` / `semantic_memory.md` — MEMORY_LOG 记录代谢流水；两个候选池文件存储当前条目
- `ITERATION_LOG.md` — 架构/技能/协议/人格变更走 ITERATION_LOG（版本化详细条目），MEMORY_LOG 只管记忆代谢
- `LTM.md` — 当前处境快照 + 跨周工作节点 + 周工作复盘走 LTM，MEMORY_LOG 只管记忆系统自身

## 关于此文件

**记忆日志** 只追踪 **候选池自身的代谢** —— 程序层/语义层条目的新增/升星/降星/毕业/删除，以及每周的记忆系统复盘。情景层（`_本周.md` / `LTM.md` / 项目主文档）不进此日志。

**何时写入：**
- 任何会话中发生候选池变更（新增、升星、降星、毕业、删除）→ 追加一行到 `§操作日志` 尾部
- 周复盘完成后 → 追加完整周复盘条目到 `§周复盘` 节尾部

**不写入：**
- 系统架构变更 / 技能创建重写 / 协议迭代 / 人格文件变动 → 走 `ITERATION_LOG.md`
- 每次对话的工作进展 → 写 `_本周.md`
- 跨周工作节点 / 周工作复盘 → 写 `LTM.md`（对应 W## 详细周录）
- 情景层条目 → 项目主文档 + `_本周.md`

**格式：**
- 操作日志：一行简短描述（`YYYY-MM-DD | [类型]：[简要描述]`），**不加 ## 前缀**
- 周复盘：三级标题（`### YYYY-WNN 周复盘（YYYY-MM-DD）`），模板见下方

**AI 何时读：**
- 启动序列尾部抽样（如最近 40 行），了解记忆系统近期节奏
- 审查代谢历史、查看上次衰减处理、确认某条目变更时间
- 读尾部 N 行即最新状态（规则前置 + 数据从旧到新追加）

## 机制要点（v3 schema prediction）

详细规则见 `CLAUDE.md §M` + `00.memory_agent.md`，此处是速查：

| 机制 | 说明 |
|---|---|
| 入池 | High（命中+偏离）→★★★；Medium（未命中）→★；Low（命中+符合）→不写 |
| 升星 | 同方向再次观察 → 星级 +1 |
| 降星 / dormant | weekly-review 扫描：连续 N 周无命中 → 降一星；0 星后再无命中 → 删除 |
| 毕业（★★★★★） | 程序层 → `SOUL/persona/persona_SOUL.md`；语义层 → `USER/USER.md`；必须经 weekly-review 确认 + 跨周稳定 |
| 冲突 | 新条目与已有矛盾 → N 级告知用户，不静默覆盖 |

## 条目模板

### 操作日志（一行写完）

```

YYYY-MM-DD | 语义层: +N条（条目名C）
YYYY-MM-DD | 记忆修正：[条目名] [原值] → [新值]
YYYY-MM-DD | 记忆删除：[条目名]（原因）
YYYY-MM-DD | 升星：[条目名] ★→★★
YYYY-MM-DD | 降星：[条目名] ★★→★
YYYY-MM-DD | 毕业：[条目名] → [目的地]
```

```
- 语义层新增：[条目名称]（如无则「无」）
- 情景→schema 提取：[本周从情景层提取升级为候选池条目的]（如无则「无」）
- 修正：[条目名称 原→新]（如无则「无」）
- 删除：[条目名称及原因]（如无则「无」）
- 毕业：[条目名称及目的地]（如无则「无」）

**强度变化**：
- 升星：[条目名称 ★→★★ 等]（如无则「无」）
- 降星：[条目名称 ★★→★ 等]（如无则「无」）

**衰减处理**：[处理了哪些连续无命中的条目，或「无」]

**复盘发现**：[本周记忆系统运行中发现的问题、模式、改进点，或「无」]

**下周迭代行动**：[记忆系统层面下周要做的调整，或「无」]

**参数调整**：[衰减周期/惊奇度判定/毕业阈值等参数变动，或「无」]
```

## 使用方法

### 工作日
- 你不用管这个文件
- AI 启动时读尾部 N 行了解近期节奏；按需读来检查某条目的变更历史

### 周末复盘
- 发起回顾对话，AI 根据本周代谢变更汇总
- 操作流水与周复盘由 AI 自动写入（S 级），事后可审查

### 下一周
- AI 按需查询上次衰减处理结果或某条目的变更时间
