## 关于此文件

**记忆日志**只追踪**候选池自身的代谢**——程序层/语义层条目的新增/升星/毕业/删除，以及每周的记忆系统复盘。情景层（`_本周.md` / `长期记忆.md` / 项目主文档）不进此日志。

**何时写入：**
- 任何会话中发生候选池变更（新增、升星、修正、毕业、删除）→ 追加一行到 `§操作日志` 尾部
- close-node 识别到未达即时入池标准、但值得周级复审的 schema 信号 → 写入 `§周复盘暂存`
- 周复盘完成后 → 追加完整周复盘条目到 `§周复盘` 节尾部，并清理已处理的暂存项

**不写入：**
- 系统架构变更 / 技能创建重写 / 协议迭代 / 人格文件变动 → 走 `ITERATION_LOG.md`
- 每次对话的工作进展 → 写 `_本周.md`
- 跨周工作节点 / 周工作复盘 → 写 `长期记忆.md`（对应 W## 详细周录）
- 情景层条目 → 项目主文档 + `_本周.md`

## 机制要点（v3.2 schema prediction）

详细规则见 `CLAUDE.md §M` + `00.memory_agent.md`，此处是速查：

| 机制 | 说明 |
|---|---|
| 入池 | High（命中+偏离）→★★★；Medium（未命中）→★；Low（命中+符合）→不写 |
| 升星 | 命中候选池 + 符合预期 → 升星（稳定性证据，不新写） |
| 修正 | 命中候选池 + 偏离预期 → 修正情境边界 / 新增例外 / 拆条 + 升星 |
| 毕业（★★★★★） | 程序层 → `SOUL/persona/persona_SOUL.md`；语义层 → `USER/USER.md`；必须经 weekly-review 确认 + 跨周稳定 |
| 暂存 | progress 段间转换中未达 schema 标准但值得复审的信号，进入 `§周复盘暂存` |
| 冲突 | 新条目与已有矛盾 → N 级告知用户，不静默覆盖 |
