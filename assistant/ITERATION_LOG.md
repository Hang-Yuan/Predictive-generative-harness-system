# Agent 架构变更日志

## 加载链

**上游：**
权威源。日常写入（daily-review 步骤 6b）与周复盘时（weekly-review 步骤 7.5）均可追加。重大变更可即时追加，不必等周复盘。

**同级联动：**
- `长期记忆.md` — 当前处境 + 跨周工作节点 + 周工作复盘走长期记忆，ITERATION_LOG 不记录工作进展
- `MEMORY/MEMORY_LOG.md` — MEMORY_LOG 只管记忆系统代谢（候选池变更 + 记忆周复盘），ITERATION_LOG 管所有架构级变更，无重叠

---

## 关于此文件

**迭代日志** 追踪 Agent 系统本身的实质性变更——不是工作内容，而是 AI 怎么思考、新学了什么技能、工作流调整、参数变化、经验教训。

**记什么（权威源，所有架构级变更都在此）：**
- 系统架构变更（文件结构 / 协议规则 / 继承链 / 加载链）
- 技能创建 / 重写 / 废弃（如「skill: daily-review 已添加」）
- 工作流变更（如「daily-review 步骤 1 加入去重规则」）
- 参数调整（如「衰减周期：4 周 → 3 周」「毕业阈值：★★★ → ★★★★★」）
- 人格文件变动（新增人格 / persona_SOUL.md 大幅重写 / 档案结构调整）
- 记忆系统协议迭代（触发规则 / 写入路由 / 记忆模板）
- Hook 脚本重写
- 重要文档大版本更新

**不记什么：**
- 每日工作进展（在 `_本周.md` 进展记录里）
- 跨周工作节点（在 `长期记忆.md` 详细周录里）
- 候选池条目变更（在 `MEMORY_LOG.md §操作日志` 里）
- 记忆周复盘（在 `MEMORY_LOG.md §周复盘` 里）
- 一般理论研究笔记

## 版本号规则

语义版本：`v主版本.次版本.补丁`

- **主版本**（X.0.0）：架构或核心工作流的破坏性变更
  - 示例：从两阶段改为三阶段周工作流
- **次版本**（0.X.0）：新功能或技能、参数调整、工作流扩展
  - 示例：添加日历生成技能
- **补丁**（0.0.X）：bug 修复、文档澄清、小优化
  - 示例：修复 MEMORY_LOG 解析

## 变更类型

| 类型 | 说明 | 示例 |
|------|------|------|
| **架构** | 系统结构变更 | 新层级、重命名区块、新工作流阶段 |
| **技能+** | 新技能创建 | 日历生成、记忆复盘、周归档 |
| **技能~** | 技能更新或废弃 | 更新记忆复盘提示、退役某技能 |
| **优化** | 性能或体验改进 | 更快的文件搜索、更清晰的提示 |
| **文档** | 文档更新（无代码变更） | 澄清区域规则、添加示例 |
| **集成** | 外部工具连接 | 对接 Obsidian、Google Calendar 等 |

## 条目格式

**最新条目在最底部**（从旧到新，读尾部即最新）。每条包含摘要、详情、文件变更、触发原因和关联链接。

```
### vX.Y.Z · YYYY-MM-DD · [类型: 架构/技能+/技能~/优化/文档/集成]

**摘要**：一句话说明改了什么

**详情**：
- 变更 1：说明
- 变更 2：说明
- （包含这个变更为什么重要）

**影响**：
| 文件 | 版本变化 | 操作 |
|------|----------|------|
| [相对路径/文件.md] | [旧 → 新] | 修改 / 新增 / 移除 |

**触发**：什么促成了这次变更
- 用户请求？bug 发现？实验结果？扩展需要？

**关联**：相关讨论、早期版本或文档的链接

---
```

## 追踪机制

### 更新时机

- **日常**（daily-review skill 步骤 6b）：会话中发生架构级变更，会话结束前追加
- **周复盘**（weekly-review skill 步骤 7.5）：如一周内有遗漏的架构变更，在周复盘时补入
- **即时记录**：重大变更发生时立即追加（不等 daily-review）

### 谁可以添加

- **AI**（运行 daily-review / weekly-review 或实施系统变更时）
- **你**（随时记录你的实验或尝试）
