## 项目间共享规则

### 跨项目引用

| 情形 | 处理 |
|---|---|
| 默认 | **默认显式隔离**，不跨引用 |
| 需要互通 | 用户明确授权后开放；同时在双方 `_overview.md §相关项目` 建立指针；登记到 §跨项目授权索引 |
| 客户交付 / 保密项目 | 在该项目 `_overview.md` YAML 加 `isolation: true`，严格不跨引用 |
| 同一结论影响多个项目 | 权威落点写一份，其他项目加指针（C 级确认） |

### 项目生命周期

```
创建 → 活跃 → 维护 / 休眠 → 收尾 → 归档
```

- **创建**：调用 `create-project` skill；同步 portfolio
- **活跃**：日常推进
- **维护**：基础已成形，按需修补，不主动迭代
- **休眠**：长时间未推进但未结项；portfolio 标 `休眠`
- **收尾**：整理产出清单，确认无遗留
- **归档**：见下

### 归档动作（详细）

1. **整理产出清单**：列出该项目所有核心产出文件（主文档 / 关键研究记录 / 文献索引）
2. **确认无未完成**：与用户对齐——是否还有悬置点 / 待解问题需要单独迁出到其他项目
3. **整体移入 `_archive/`**：保持项目目录结构完整迁移
4. **标只读**：归档后只读不写；如需翻阅或引用，原样保留
5. **永不删除**：归档替代删除；除非用户显式要求删除（C 级）
6. **同步 portfolio**：状态字段改为 `已归档`，最近节点写"结项 + 一句话总结"

## 项目内部结构规范（最小公共单元）

> 本节约束所有项目 `_overview.md` 层之下的结构。agent 只管到 _overview.md，再往下由各项目 _overview.md §项目文件 自声明——这是封臣界线。

**完整项目结构**（按需建，不预建空目录）：

```
01 项目区/[项目名]/
├── _overview.md              ← 项目入口（必建）
├── [项目主文档].md            ← 项目核心产出（按需）
├── _progress/                ← 推进记录容器（按需）
│   ├── [推进管家文件].md      ← 推论链（节点骨架 + 指针）
│   └── [推进展开子文件].md    ← 推进展开（具体推论 / 证据 / 分支）
├── _reference/               ← 外部素材容器（按需）
│   └── 文献记录.md            ← 文献引用索引（科研类）
└── _archive/                 ← 归档容器（按需）
    └── [旧版本主文档].md
```

**各部分职能**：

| 名称 | 职能 | 何时建 |
|---|---|---|
| `_overview.md` | 项目入口；顶层状态 / 文件清单 / 跨项目引用 | create-project skill 建 |
| `[项目主文档].md` | 项目核心产出（理论 draft / 课程大纲 / 商业方案等） | 第一个实质内容产出时 |
| `_progress/` | 推进记录容器 | 首次需要推进记录时由 write-progress skill 建 |
| `_progress/[推进管家文件].md` | 推论链：节点骨架 + 指针。命名按项目类型：理论研究 = `_research_notes.md`；课程 = `_course_notes.md`；事务 = `_logs.md`；商业 = `_decisions.md`；系统工程 = `_system_notes.md` | 同上 |
| `_progress/[推进展开子文件].md` | 推进展开：具体推论 / 证据 / 分支 | 推进树节点引出新主题时 |
| `_reference/` | 外部素材 / 文献容器 | 首次需要文献时由 manage-research-reference skill 建（待建） |
| `_archive/` | 归档旧版本 / 历史快照 | 首次归档时 |

**命名约定**（骨架层英文，内容层中文）：

- **骨架文件与功能性目录**全英文下划线命名：`_overview.md` / `_archive/` / `_reference/` / `_progress/`
- **推进管家文件**英文下划线命名，按项目类型语义命：`_research_notes.md` / `_course_notes.md` / `_logs.md` / `_decisions.md` / `_system_notes.md`
- **内容文件**（主文档 / 领域性命名）可保留中文或英文
- **项目目录名**优先英文；纯业务中文场景可保留中文

**语义分工**：
- `_progress/` 文件夹名 = "进展"功能层（通用，所有项目类型相同）
- 容器内推进管家文件名 = 内容性质特化（研究笔记 / 课程笔记 / 事务日志 / 商业决策……按项目类型选）
- 二者解耦：文件夹名不变，管家文件按类型挂不同语义牌

**推进容器内部写作规范**（节点格式 / 推进树和推进展开的组织原则 / 追加规则）见 `write-progress` skill。本 agent 不直接定义。
